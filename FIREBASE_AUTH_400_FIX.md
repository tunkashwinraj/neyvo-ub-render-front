# Fix 400 errors on Login / Sign up (Firebase Identity Toolkit)

When you see **400** from `identitytoolkit.googleapis.com/v1/accounts:signInWithPassword` or `signUp`, the request is being rejected by Firebase. Fix it in the **Firebase Console** (and optionally in your project config).

---

## 1. Enable Email/Password sign-in

1. Open [Firebase Console](https://console.firebase.google.com/) and select your project (e.g. **neyvo-launch**).
2. Go to **Authentication** → **Sign-in method**.
3. Click **Email/Password**.
4. Turn **Enable** ON and save.

If this is off, Firebase returns 400 for sign-in and sign-up.

---

## 2. Add your app origin to Authorized domains

1. In the same project: **Authentication** → **Settings** (or **Authorized domains** tab).
2. Under **Authorized domains**, ensure:
   - **localhost** is listed (for `flutter run -d chrome` and local dev).
   - Your production domain is listed when you deploy (e.g. `yourapp.web.app` or your custom domain).

If the domain is missing, Firebase can reject the request with 400.

---

## 3. Use the correct Firebase project

Your app uses the project defined in **lib/firebase_options.dart** (projectId, apiKey, etc.). That file is generated by the **FlutterFire CLI**.

- If you created a **new** Firebase project for Neyvo Pulse, regenerate options for that project and replace `firebase_options.dart`:

  ```bash
  cd neyvo_pulse_front
  dart run flutterfire configure
  ```

  Then pick the new project and re-run the app.

- If you use the **existing** project (e.g. `neyvo-launch`), complete steps 1 and 2 above for that project.

---

## 4. Check Identity Toolkit API (Google Cloud)

1. Open [Google Cloud Console](https://console.cloud.google.com/) and select the **same** project as in Firebase (e.g. neyvo-launch).
2. Go to **APIs & Services** → **Enabled APIs**.
3. Ensure **Identity Toolkit API** (or **Firebase Authentication API**) is enabled. If not, enable it.

---

## 5. Password rules

- Firebase requires **at least 6 characters** for password. The app now validates this before calling Firebase so you see a clear message instead of a 400.

---

## Summary checklist

| Check | Where |
|-------|--------|
| Email/Password provider **Enabled** | Firebase Console → Authentication → Sign-in method |
| **localhost** in Authorized domains | Authentication → Settings → Authorized domains |
| **Identity Toolkit API** enabled | Google Cloud Console → APIs & Services |
| Correct project in `firebase_options.dart` | Run `dart run flutterfire configure` if you switched project |

After changing these, try sign-in or sign-up again. If it still returns 400, open the **Network** tab in DevTools, click the failed `signInWithPassword` or `signUp` request, and check the **Response** body—Firebase often returns a short reason (e.g. `INVALID_EMAIL`, `WEAK_PASSWORD`, or a config error).
